## 1. 가설

신규 perp 상장 직후 윈도우에 **초기 과열(과잉 변동)의 평균회귀** 패턴이
반복된다. 자산에 과거 데이터가 없어 퀀트펀드가 모델링 못 하는 시간한정
비효율 → 룰베이스로 방향 예측 없이 추출 가능.

핵심 테스트(비예측·기계적):
- **H1 (주력) 초기 스파이크 페이드**: 상장+`E`분 시점, open 대비 |이동| >
  `T`% 면 그 반대로 진입, `H`분 보유 또는 SL/TP. (실현 이동에 대한 기계적 룰)
- **H2 (대조) 초기 레인지 돌파 추종**: 첫 `E`분 레인지 돌파 방향 추종.

H1·H2는 같은 데이터로 동시 측정(엣지가 페이드인지 모멘텀인지, 아니면 둘 다
무인지 판별).

## 2. 데이터

- 소스: Binance fapi (수집 스크립트 신규, 봇/src 무변경)
- 대상: 최근 24개월 상장 USDT PERP (N≈324, 생존). BTC/ETH 등 메이저 재상장
  성격 아닌 *신규 토큰*만(onboardDate 기준 신규, 첫 봉 존재)
- 봉: 상장시점부터 1m, 최소 첫 24h(1440봉). 캐시(`data/listings/{sym}.parquet`)
- 이벤트 정렬: t0 = onboardDate(1st bar open)

## 3. 비용 모델 (보수적 — 신규상장은 슬리피지가 치명)

신규 상장 토큰의 초기 스프레드/슬리피지는 메이저와 차원이 다름. **공격적으로
보수 가정**:
- taker fee 0.04%/leg
- 슬리피지 시나리오 매트릭스: {0.3%, 0.6%, 1.0%} per fill (메이저 0.01%가
  아님 — 이 가정이 결과를 지배할 것)
- 진입+청산 = 2 fill
- 결과는 슬리피지 3종 전부 리포트. 보수(1.0%)에서 net+가 아니면 신뢰 불가

## 4. 메트릭 (이벤트 스터디 — 방향 PF 아님)

| 메트릭 | 의미 | 기준 |
|--------|------|------|
| 이벤트당 net 평균수익(bps) | 비용 후 1상장당 기대 | >0, 슬립1.0%서도 >0 목표 |
| 적중률 | net>0 이벤트 비율 | >50% |
| 중앙값 net | 아웃라이어 의존도 점검 | >0 (평균≫중앙값이면 소수 대박 의존=취약) |
| 분포/표준편차·Sharpe류 | 안정성 | 평균/표준편차 의미있게 + |
| 연도별 일관성 | 2024 vs 2025 vs 2026 | 모든 연도 동일 부호 |
| tradable N | 룰 충족 이벤트 수 | >30 (실측 N 큼, 충분 예상) |

## 5. 사전 폐기 기준 (실행 전 확정 — 변경 금지)

**하나라도 해당 시 즉시 FAIL → 폐기 → from-scratch 재설계 복귀:**

1. 보수 슬리피지(1.0%) 기준 이벤트당 net 평균 ≤ 0
2. 중앙값 net ≤ 0 이면서 평균만 + (소수 아웃라이어 의존 = 실거래 불가능한
   취약 엣지)
3. 연도 부호 불일치(예: 2024 +, 2025 −) — 비반복 패턴
4. 적중률 < 50% 이면서 평균도 미미
5. H1·H2 둘 다 net ≤ 0 (페이드도 모멘텀도 무 → 패턴 부재)

**PASS (다음 단계 = 실행가능성 설계, 봇코드 아님):**
위 전부 통과 + 슬립 0.6%서 이벤트당 net 명백히 >0 + 연도 일관 + 중앙값도 +.
PASS여도 생존편향 할인 명시 + 실행지연(상장 직후 봇이 실제 체결 가능한가)
별도 검증 전까지 봇 미연결.

## 6. 백테스트 설계

   판정 안 함)
3. 결과 → `2026-05-17-new-listing-microstructure-result.md`, CLAUDE.md/메모리
