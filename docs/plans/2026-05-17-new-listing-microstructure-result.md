# New-Listing Microstructure — 백테스트 결과

> 작성일: 2026-05-17 · 상태: **FAIL — 폐기 (10전 10패)** · 설계: `2026-05-17-new-listing-microstructure-design.md`

## 가설

신규 perp 상장 직후 초기 과열의 평균회귀(H1 페이드) 또는 레인지 돌파
추종(H2)이 반복적·기계적으로 추출 가능한가. 비예측 룰베이스.

## 데이터

Binance fapi: 최근 24개월 신규 USDT PERP **N=323 상장** (2024:72,
2025:205, 2026:35 부근), 상장시점부터 1m 25h klines. **생존편향**: 현재
거래중 심볼만 → 실패/폐지 상장 제외, 엣지 *과대* 방향.

## 결과 (사전 폐기기준 §5 대조)

### H1 페이드 — 전체 N에서 붕괴
- 최우수 그리드(E=30 T=20% H=30): slip 0.6% 평균 **-2.6bps**, slip 1.0%
  **-82.6bps**. 전 그리드 셀이 slip 0.6%서 ≤0.
- 연도 부호 불일치: 2024 −, 2025 ±, 2026 + (슬립/연도마다 부호 뒤집힘).
- **부분데이터(95건) "힌트"는 허상이었다**: 당시 최우수 E60/T10/H240
  +275bps(n=31)는 2024-편향 저N 서브샘플 아티팩트 → 전체 N에서 top8에도
  없음. 과적합·체리픽의 교과서 사례 (사전기준+전체데이터 규율이 잡아냄).

### H2 돌파 — 아웃라이어 의존, 비용서 음전환
- 최우수(E=5 H=240): slip 0.6% 평균 +44.1bps이나 **중앙값 -97.8bps** —
  소수 대박 의존, 전형적 사건은 손실(실거래 불가 취약 엣지).
- slip 1.0%서 평균 **-35.9bps**, 적중률 43~48%(<50%), 연도부호 불일치.

### 폐기기준 위반 (다중)
1. 보수 슬립(1.0%) 이벤트 net ≤ 0 — H1·H2 모두 → KILL
2. 중앙값 ≤0 + 평균만 + (아웃라이어 의존) — H2 명백 → KILL
3. 연도 부호 불일치 — H1·H2 모두 → KILL
4. 적중률 <50% + 평균 미미 — H2 → KILL
5. H1·H2 둘 다 보수비용서 net ≤0 (페이드도 모멘텀도 무 = 패턴 부재) → KILL

추가로 위 전부 **생존편향 할인 적용 전** 수치 — 보정하면 더 악화.

## 결론: **FAIL — 폐기. 10전 10패.**

"입력을 바꾼다"(정보/구조 비대칭) 경로 중 유일하게 backtest 가능했던
신규상장 마이크로구조도, 비용·생존편향·과적합을 직시하면 net 엣지 없음.
부분데이터 힌트가 전체 N에서 소멸한 것은 *규율(사전기준+전체표본)이
의도대로 작동*해 값비싼 실거래 실패를 **싸게(1일·무자본) 차단**한 사례.

## 프로세스 관점 (중요)

이건 알파로는 10번째 실패지만, *프로세스로는 성공*이다: "입력을 바꾸면
다를까"라는 재고를 **자본 위험 0으로 하루 만에 falsify**했다. 느리고
비싼 NO 대신 빠르고 싼 NO를 얻었다.

## 에스컬레이션

사용자는 9패 시점에 이미 **"인프라/학습 자산으로 전환"** 을 선택했고,
그 후 "입력 변경"을 재고했다. 본 결과로 그 재고도 falsified →
**원래 선택(인프라/학습 자산)이 증거상 옳았음이 재확인됨.** incremental
신호(#11) 금지. 알파 추구 라인 종결, 비-알파 가치(엔지니어링·학습·리스크
도구)로 전환이 합리적 결론.
